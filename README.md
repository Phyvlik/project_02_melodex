# Melodex

Collaborative music queue app built with Flutter and Firebase. Groups create or join a shared room, search for songs via the Spotify Web API, and vote democratically on what plays next. A scoring engine combines vote counts, room mood, and listen history to surface the three best "Up Next" suggestions with transparent reasoning and manual override controls.

## Features

- **Authentication** -- Email/password sign-in and registration via Firebase Auth, with persistent sessions
- **Rooms** -- Create a named room with a mood tag, or join one with a 6-character invite code
- **Collaborative Queue** -- Real-time playlist synced across all members using Firestore streams
- **Democratic Voting** -- Upvote or downvote any song; vote counts updated atomically with Firestore transactions
- **Spotify Search** -- Search 80M+ tracks by title or artist using the Spotify Web API (client credentials flow)
- **Live Chat** -- Room-scoped message thread with system messages for song adds and queue changes
- **AI Recommendations** -- Scoring engine ranks next songs by votes (50%), mood match (30%), and genre history (20%), with inline score breakdown and host override controls
- **Push Notifications** -- FCM topic-based alerts when the queue is empty or a new song is added
- **Profile** -- Display name and photo (photo upload requires Firebase Storage / Blaze plan)
- **Dark Theme** -- Material 3 dark theme with Spotify green (#1DB954) as the primary accent

## AI Challenge: Recommendation Scoring

The `RecommendationService` scores every song in the queue using three signals:

| Signal | Weight | Logic |
|--------|--------|-------|
| Vote score | 50% | `(upvotes - downvotes)` normalized 0.0 to 1.0 across the queue |
| Mood match | 30% | 1.0 for exact match, 0.5 if song is untagged, 0.0 for mismatch |
| Genre history | 20% | +0.1 per matching genre in user listen history, -0.3 if played in last 10 songs |

The top three results are shown in the **Up Next** tab with a visible score breakdown (e.g., "Votes 0.8 + Mood 1.0 + Genre 0.6") so users can see exactly why a song was ranked. The host can pin or skip any suggestion using the override controls.

## Security Rules

Firestore rules enforce row-level security:

- Any signed-in user can read room listings and create a room
- Only room members (`memberUids` array) can read/write the playlist, votes, and messages subcollections
- Only the host can delete rooms or remove songs from the queue
- Users can only write their own vote document (`request.auth.uid == request.resource.data.userId`)

Rules are deployed from `firestore.rules`. To redeploy:

```bash
firebase deploy --only firestore:rules
```

## Getting Started

### Prerequisites

- Flutter 3.41.6+
- A Firebase project with Auth, Firestore, and FCM enabled
- A Spotify Developer app (client credentials)

### Setup

1. **Clone the repo**

```bash
git clone https://github.com/Phyvlik/project_02_melodex.git
cd project_02_melodex
```

2. **Install dependencies**

```bash
flutter pub get
```

3. **Add Firebase config**

   Place `google-services.json` (Android) inside `android/app/`. The `lib/firebase_options.dart` file is already generated and committed.

4. **Spotify credentials**

   Open `lib/services/spotify_service.dart` and set `_clientId` and `_clientSecret` from your Spotify Developer Dashboard app. (Already configured for the team's app.)

5. **Run**

```bash
flutter run
```

## Known Limitations

- Firebase Storage features (profile photos, album art caching) require upgrading the Firebase project to the Blaze pay-as-you-go plan.
- Spotify preview URLs (30-second clips) are available on some tracks only; availability depends on Spotify's licensing per market.
- The Spotify Developer app is in Development mode, so only explicitly added Spotify accounts can use song search during testing.
